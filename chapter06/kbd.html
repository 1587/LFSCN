6.51. Kbd-1.15.5

Kbd 程序包包含按键表文件、控制台字体和键盘工具。

预计编制时间：0.1 SBU
需求磁盘空间：20 MB

6.51.1. 安装 Kbd

退格键和删除键的行为在 Kbd 程序包中不同的键盘映射中是不一致的。以下补丁为 i386 键盘映射解决此问题： 

patch -Np1 -i ../kbd-1.15.5-backspace-1.patch

应用补丁后，Backspace 键生成为 127 号字符，而 Delete 键生成广为人知的转义序列。

修复一个漏洞，此漏洞会导致某些按键表不能正确加载： 

sed -i -e '326 s/if/while/' src/loadkeys.analyze.l

准备编译 Kbd ： 

./configure --prefix=/usr --datadir=/lib/kbd \
  --disable-vlock

配置选项的含义：

--datadir=/lib/kbd

此选项将键盘布局数据存置于一个总是位于根分区的目录下而非默认的 /usr/share/kbd。

--disable-vlock

此选项阻止编制 vlock 工具，它需求的 PAM 库在 chroot 环境中不可用。

编译程序包： 

make

此程序包不包含测试套件。

安装程序包： 

make install

注意

对于有些语言 (如白俄罗斯语)， Kbd 程序包在常规的 “by” 键盘映射认定 ISO-8859-5 编码的情况下无法提供有效的键盘映射，此情况下通常使用 CP1251 键盘映射。这些语言的用户需要单独下载能够工作的效键盘映射。

LFS-Bootscripts 程序包中的一些脚本依赖 kbd_mode，loadkeys，openvt，和 setfont。鉴于 /usr 在引导早期阶段中可能不可用，这些二进制程序需要位于根分区上： 

mv -v /usr/bin/{kbd_mode,loadkeys,openvt,setfont} /bin

如果需要，安装文档： 

mkdir -v /usr/share/doc/kbd-1.15.5
cp -R -v doc/* \
         /usr/share/doc/kbd-1.15.5

6.51.2. Kbd 的内容

安装的程序：chvt, deallocvt, dumpkeys, fgconsole, getkeycodes, kbdinfo, kbd_mode, kbdrate, loadkeys, loadunimap, mapscrn, openvt, psfaddtable (到 psfxtable 的链接), psfgettable (到 psfxtable 的链接), psfstriptable (到 psfxtable 的链接), psfxtable, setfont, setkeycodes, setleds, setmetamode, showconsolefont, showkey, unicode_start 和 unicode_stop
安装的目录：/lib/kbd

简介

chvt

改变前景的虚拟终端

deallocvt

解除分配未使用的虚拟终端

dumpkeys

转储键盘翻译表

fgconsole

打印活动的虚拟终端数

getkeycodes

打印内核 scancode-to-keycode 映射表

kbdinfo

获取一个控制台的状态信息

kbd_mode

报告或设置键盘模式

kbdrate

设置键盘重复和延迟率

loadkeys

加载键盘翻译表

loadunimap

加载内核 unicode-to-font 映射表

mapscrn

一个弃用的程序，曾用于向控制台驱动程序中加载用户定义的输出字符映射表；此工作现在由 setfont 完成

openvt

在一个新的虚拟终端 (VT) 上启动一个程序

psfaddtable

到 psfxtable 的链接

psfgettable

到 psfxtable 的链接

psfstriptable

到 psfxtable 的链接

psfxtable

为控制台字体处理 Unicode 字符表

setfont

改变控制台上的 EGA 和 VGA 字体

setkeycodes

加载核心 scancode-to-keycode 映射表条目；这键盘上存在非常规按键时很有用

setleds

设置键盘上的标识和 LED

setmetamode

定义键盘的 meta 按键处理

showconsolefont

显示当前 EGA/VGA 控制台屏幕字体

showkey

报告键盘上按下的键的扫描码、按键码和 ASCII 码

unicode_start

令键盘和控制台为 UNICODE 模式。除非你的键盘映射文件是 ISO-8859-1 编码，不要使用此程序。对于其他编码，此工具会产生错误的结果

unicode_stop

将键盘和控制台从 UNICODE 模式还原
