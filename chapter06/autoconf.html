6.38. Autoconf-2.69

Autoconf 程序包包含用于产生可以自动配置源代码的 shell 脚本的程序。

预计编制时间：4.5 SBU
需求磁盘空间：17.1 MB

6.38.1. 安装 Autoconf

准备编译 Autoconf： 

./configure --prefix=/usr

编译程序包： 

make

要测试结果，执行： 

make check

这一步会占用很长时间，大约要 4.7 SBU，而且有 6 个使用 Automake 的测试被跳过了。要完整覆盖之，可以在 Automake 安装之后重新测试。

安装程序包： 

make install

6.38.2. Autoconf 的内容

安装的程序：autoconf, autoheader, autom4te, autoreconf, autoscan, autoupdate 和 ifnames
安装的目录：/usr/share/autoconf

简介

autoconf

产生可以自动配置软件源代码包以移植到多种类 Unix 系统上；产生的脚本是独立的——运行时不再需要 autoconf 程序

autoheader

用于创建 C 语言的 #define 声明的模板文件供 configure 使用

autom4te

M4 宏处理器的一个打包

autoreconf

自动按顺序运行 autoconf、autoheader、aclocal、automake、gettextize 和 libtoolize，在 autoconf 和 automake 的模板文件有改动时可以节省时间

autoscan

协助给程序包创建一个 configure.in 文件；它检查目录树中的源代码文件，搜索常见可移植性问题，然后创建一个 configure.scan 文件作为程序包预备的 configure.in 文件

autoupdate

修改仍然用旧的宏调用 autoconf 的 configure.in 文件，使其使用新的宏名称

ifnames

协助编写程序包的 configure.in 文件；它打印出程序在 C 预处理条件中使用的标识符；如果一个程序包已经设置为有一定的可移植性，此程序可以协助分析 configure 需要检查什么。它还可以填满 autoscan 出样建的 configure.in 文件中的空格
