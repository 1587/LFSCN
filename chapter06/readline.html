6.32. Readline-6.2

Readline 程序包是一系列库，提供命令行编辑和历史兼容性。

预计编制时间：0.1 SBU
需求磁盘空间：17.2 MB

6.32.1. 安装 Readline

重新安装 Readline 会导致旧的库被移动到 <libraryname>.old。这通常不是问题，但是可能导致 ldconfig 的一个链接问题。执行以下两次 sed 防止此问题出现： 

sed -i '/MV.*old/d' Makefile.in
sed -i '/{OLDSUFF}/c:' support/shlib-install

应用一个补丁修复一个在上游被修复了的已知漏洞： 

patch -Np1 -i ../readline-6.2-fixes-1.patch

准备编译 Readline： 

./configure --prefix=/usr --libdir=/lib

编译程序包： 

make SHLIB_LIBS=-lncurses

make 选项的含义：

SHLIB_LIBS=-lncurses

此选项强制 Readline 链接至 libncurses (实际上是 libncursesw) 库。

此程序包不包含测试套件。

安装程序包： 

make install

现在将静态库移动到更合适的位置： 

mv -v /lib/lib{readline,history}.a /usr/lib

下一步移动 /lib 中的 .so 文件并重新链接到 /usr/lib： 

rm -v /lib/lib{readline,history}.so
ln -sfv ../../lib/libreadline.so.6 /usr/lib/libreadline.so
ln -sfv ../../lib/libhistory.so.6 /usr/lib/libhistory.so

如果需要，安装文档： 

mkdir   -v       /usr/share/doc/readline-6.2
install -v -m644 doc/*.{ps,pdf,html,dvi} \
                 /usr/share/doc/readline-6.2

6.32.2. Readline 的内容

安装的库：libhistory.{a,so} 和 libreadline.{a,so}
安装的目录：/usr/include/readline、/usr/share/readline、/usr/share/doc/readline-6.2

简介

libhistory

提供召回命令行历史的统一用户界面

libreadline

对分立的需要提供命令行界面的程序提升用户界面的统一性
