6.33. Bash-4.2

Bash 程序包包含 Bourne-Again SHell。

预计编制时间：1.7 SBU
需求磁盘空间：45 MB

6.33.1. 安装 Bash

首先应用以下补丁修复上游指出的多个漏洞： 

patch -Np1 -i ../bash-4.2-fixes-9.patch

准备编译 Bash： 

./configure --prefix=/usr                     \
            --bindir=/bin                     \
            --htmldir=/usr/share/doc/bash-4.2 \
            --without-bash-malloc             \
            --with-installed-readline

配置选项的含义：

--htmldir

此选项指定 HTML 格式的文档的安装目录。

--with-installed-readline

此选项告诉 Bash 使用系统上已经安装的 readline 库而不使用自己的 readline 版本。

编译程序包： 

make

如果不运行测试套件那么跳到 “安装程序包”。

要准备测试，确保 nobody 用户可以写入源代码树： 

chown -Rv nobody .

现在，作为 nobody 用户运行测试： 

su nobody -s /bin/bash -c "PATH=$PATH make tests"

安装程序包： 

make install

运行新编译的 bash 程序 (替换掉现在在运行的副本)： 

exec /bin/bash --login +h

注意

这些选项使 bash 进程成为一个交互的登录 shell 并且继续禁用哈希使得新的程序一旦被可用就会立刻被发现。

6.33.2. Bash 的内容

安装的程序：bash、bashbug 和 sh (到 bash 的链接)
安装的目录：/usr/share/doc/bash-4.2

简介

bash

一个广泛使用的翻译器；在运行命令之前对其进行很多扩展和替换，使其成为一个强大的工具

bashbug

一个 shell 脚本，帮助用户组织和发送标准格式的有关 bash 的漏洞报告

sh

到 bash 程序的符号链接；被作为 sh 调用时 bash 试图尽可能模仿 sh 历史版本的行为，并且符合 POSIX 标准的要求
