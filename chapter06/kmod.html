6.52. Kmod-12

Kmod 程序包包含用于加载内核模块的库和工具

预计编制时间：0.1 SBU
需求磁盘空间：30 MB

6.52.1. 安装 Kmod

准备编译 Kmod： 

./configure --prefix=/usr       \
            --bindir=/bin       \
            --libdir=/lib       \
            --sysconfdir=/etc   \
            --disable-manpages  \
            --with-xz           \
            --with-zlib

配置选项的含义：

--with-*

这些选项允许 Kmod 处理压缩的内核模块。这些 shell 变量用于在 pkg-config 缺失时使 xz 和 zlib 头文件能够被找到。

--disable-manpages

这一选项防止手册页被编制，因为它们依赖于在 chroot 环境下不可用的 libxsit。

编译程序包： 

make

要测试结果，执行： 

make check

安装程序包并创建符号链接以兼容 Module-Init-Tools，即先前用于处理 Linux 内核模块的程序包： 

make pkgconfigdir=/usr/lib/pkgconfig install

for target in depmod insmod modinfo modprobe rmmod; do
  ln -sv ../bin/kmod /sbin/$target
done

ln -sv kmod /bin/lsmod

6.52.2. Kmod 的内容

安装的程序：depmod (到 kmod 的链接)、insmod (到 kmod 的链接)、kmod、kmod-nolib、lsmod (到 kmod 的链接)、modinfo (到 kmod 的链接)、modprobe (到 kmod 的链接) 和 rmmod (到 kmod 的链接)
安装的库：/lib/kmod.so

简介

depmod

基于已有模块中发现的符号创建一个依赖关系文件；此文件被 modprobe 用于自动加载需要的模块

insmod

向运行中的内核安装一个可加载的模块

kmod

加载和卸载内核模块

libkmod

此库被其他程序用于加载和卸载内核模块

lsmod

列出当前加载的模块

modinfo

检查与内核模块关联的对象文件并显示收集到的信息

modprobe

使用 depmod 创建的依赖关系文件自动加载对应的模块

rmmod

从运行中的内核中卸载模块
