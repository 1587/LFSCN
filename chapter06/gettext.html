6.44. Gettext-0.18.1.1

Gettext 程序包包含用于国际化和本地化的工具。这些工具允许程序带有 NLS (原生语言支持) 编译，从而使它们能够以用户的本土语言输出消息。

预计编制时间：2.3 SBU
需求磁盘空间：180 MB

6.44.1. 安装 Gettext

修复此程序包和 Glibc-2.16.0 间的一个不兼容问题： 

sed -i -e '/gets is a/d' gettext-*/*/stdio.in.h

准备编译 Gettext ： 

./configure --prefix=/usr \
            --docdir=/usr/share/doc/gettext-0.18.1.1

编译程序包： 

make

要测试结果 (这需要较长时间，大约 3 SBUs) ，执行： 

make check

安装程序包： 

make install

6.44.2. Gettext 的内容

安装的程序：autopoint, config.charset, config.rpath, envsubst, gettext, gettext.sh, gettextize, hostname, msgattrib, msgcat, msgcmp, msgcomm, msgconv, msgen, msgexec, msgfilter, msgfmt, msggrep, msginit, msgmerge, msgunfmt, msguniq, ngettext, recode-sr-latin 和 xgettext
安装的库：libasprintf.{a,so}, libgettextlib.so, libgettextpo.{a,so}, libgettextsrc.so, and preloadable_libintl.so
安装的目录：/usr/lib/gettext, /usr/share/doc/gettext-0.18.1.1, /usr/share/gettext

简介

autopoint

复制标准 Gettext 基础文件到一个源程序包

config.charset

输出一个基于系统的字符编码镜像表

config.rpath

输出一组基于系统的变量，他们描述如何设置一个可执行文件中共享库的运行库搜索路径

envsubst

替换 shell 格式字符串中的环境变量

gettext

查阅某一信息目录来将一种自然语言翻译成用户的语言

gettext.sh

主要用作一个 gettext 的一个 shell 函数库

gettextize

复制所有标准 Gettext 文件到某一程序包下的给定顶层目录来开始其国际化进程

hostname

一不同的方式显示一个网络 host 名称

msgattrib

根据它们属性的筛选一个翻译目录中的信息，并处理这些属性

msgcat

连接并混合给定的 .po 文件

msgcmp

比较两个 .po 文件以检查它们是否包含相同的 msgid 字符串组

msgcomm

找到给定 .po 文件共有的信息

msgconv

转换一个翻译目录为一种不同的字符编码

msgen

创建一个英文翻译目录

msgexec

应用一个命令到某一翻译目录的所有翻译

msgfilter

应用一个筛选到某一翻译目录的所有翻译

msgfmt

根据一个翻译目录生成一个二进制信息目录

msggrep

提取某一符合一个给定样式或属于一些给定源文件的翻译目录中的所有信息

msginit

创建一个新的 .po 文件，用用户环境的值初始化元信息

msgmerge

组合两个原始翻译到一个文件中

msgunfmt

反编译一个二进制信息目录为原始翻译文档

msguniq

使某一翻译目录下的镜像翻译一致

ngettext

显示某一语法形式基于一个数字的原文信息的本地语言翻译

recode-sr-latin

重编码 Cyrillic 脚本的 Serbian 文本到 Latin 脚本

xgettext

提取给定源文件中可翻译的信息行来制作初次翻译样板

libasprintf

定义 autosprintf 类，它使 C 格式输出途径在 C++ 程序中可用，以使用 <string> 字符串和 <iostream> 流

libgettextlib

使一个包含通常途径的私有库被不同的 Gettext 程序使用；这并不是为了一般的用途

libgettextpo

用来编写特殊的处理 .po 文件的程序；这个库在 Gettext 包含的应用程序 （比如 msgcomm，msgcmp，msgattrib，和 msgen）不能满足时被用到

libgettextsrc

一个被不同 Gettext 程序使用到的包含通常途径的私有库；这并不是为了一般的用途

preloadable_libintl

一个被 LD_PRELOAD 用以在记录未翻译信息时辅助 libintl 的库。
