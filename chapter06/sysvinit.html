6.58. Sysvinit-2.88dsf

Sysvinit 程序包包含控制系统启动，运行和关机的程序。

预计编制时间：少于 0.1 SBU
需求磁盘空间：1.4 MB

6.58.1. 安装 Sysvinit

当运行等级改变 (例如终止系统时)， init 传递终止信号到各个由 init 本身启动且不应当在新运行等级下运行的进程。如此做时， init 输出像 “Sending processes the TERM signal” 这样的消息，似乎是暗示它正在传递这些信号到当前所有运行进程的信息。要避免这一误解，修改源文件中的信息为 “Sending processes configured via /etc/inittab the TERM signal”： 

sed -i 's@Sending processes@& configured via /etc/inittab@g' src/init.c

wall 和 mountpoint 程序正在维护的版本在此前已由 Util-linux 安装。阻止这些程序和它们的手册页的 Sysvinit 版本被安装： 

sed -i -e 's/utmpdump/d' \
       -e '/mountpoint/d' src/Makefile

编译程序包： 

make -C src

这个程序包不包含测试套件。

安装程序包： 

make -C src install

6.58.2. Sysvinit 的内容

安装的程序：bootlogd、fstab-decode、halt、init、killall5、last、lastb (到 last 的链接)、mesg、pidof (到 killall5 的链接)、poweroff (到 halt 的链接)、reboot (到 halt 的链接)、runlevel、shutdown、sulogin 和 telinit (到 init 的链接)

简介

bootlogd

记录引导信息到一个日志文件

fstab-decode

以 fstab 编码的参数运行一个命令

halt

以 -h 选项正常调用 shutdown，在系统已经处于运行等级 0 时则告知内核终止系统；它在 /var/log/wtmp 文件中记录系统正被关闭的信息

init

内核初始化硬件后首先启动的进程，用以接管引导进程和启动它指向的所有进程

killall5

传递一个信息到除在其自身会话中的进程外的所有进程，因此它不会终止运行调用它的脚本的 shell

last

检索 /var/log/wtmp 文件并显示用户最后一次登陆 (与登出) 的系统；它同样显示系统引导，关机和运行等级的改变

lastb

显示记录在 /var/log/btmp 中的失败的登陆尝试

mesg

控制其他用户能否传递信息到当前用户的终端

pidof

报告给定程序的 PIDs

poweroff

告知内核终止系统且关闭计算机 (参见 halt)

reboot

告知内核重新引导系统 (参见 halt)

runlevel

报告记录在 /var/run/utmp 中的最近一次运行等级记录中的最近和当前运行等级

shutdown

以安全途径关闭系统，通知所有进程与所有登陆的用户

sulogin

允许 root 登陆；它一般由 init 在系统进入单独用户模式时调用

telinit

通知 init 运行等级被改为何等级
