6.42. Findutils-4.4.2

Findutils 程序包包含用于查找文件的程序。这些程序递归搜索目录树并创建、维护和搜索一个数据库 (通常比递归查找快，但如果数据库最近没有更新则可能不可靠)。

预计编制时间：0.4 SBU
需求磁盘空间：29 MB

6.42.1. 安装 Findutils

准备编译 Findutils： 

./configure --prefix=/usr                   \
            --libexecdir=/usr/lib/findutils \
            --localstatedir=/var/lib/locate

配置选项的含义：

--localstatedir

此选项改变 locate 数据库的位置为 /var/lib/locate，这个位置与 FHS 兼容。

编译程序包： 

make

要测试结果，执行： 

make check

安装程序包： 

make install

LFS-bootscripts 中的某些脚本依赖 find。由于 /usr 在引导早期可能还不可用，此程序需要在根分区上。updatedb 脚本也需要修改才能修正这个路径： 

mv -v /usr/bin/find /bin
sed -i 's/find:=${BINDIR}/find:=\/bin/' /usr/bin/updatedb

6.42.2. Findutils 的内容

安装的程序：bigram, code, find, frcode, locate, oldfind, updatedb 和 xargs
安装的目录：/usr/lib/findutils

简介

bigram

曾用于产生 locate 数据库

code

曾用于产生 locate 数据库；是 frcode 的前身

find

搜索给定的目录树，查找匹配指定条件的文件

frcode

被 updatedb 调用，用于压缩文件名列表；使用前向压缩将数据库尺寸缩减到 1/5 到 1/4

locate

搜索一个文件名数据库，报告包含给定字串或匹配给定样式的名称

oldfind

旧版本的 find；使用的是一种不同的算法

updatedb

更新 locate 数据库；扫描整个文件系统 (默认包含当前挂载的其他文件系统) 并把所有文件名放入数据库

xargs

可以用于对多个文件应用指定的命令
