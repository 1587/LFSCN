5.5. GCC-4.6.3 - 第一遍

GCC 程序包包含 GNU 编译器集，其中包括 C 和 C++ 编译器。

预计编制时间: 5.0 SBU
需求磁盘空间: 1.2 GB

5.5.1. 安装交叉 GCC

GCC 现在需求 GMP、MPFR 和 MPC 程序包。由于这些程序包可能未包含与你的宿主发行版中，它们将与 GCC 一同编制。将每个程序包解包至 GCC 源代码目录并重命名产生的文件夹使得 GCC 编制过程能够自动使用他们：

注意

关于这一节经常出现误解。此过程与其他每一节都是一样的，就像之前在程序包编制指令中解释的那样。首先从源代码目录中提取 gcc 压缩包然后改变到创建的目录，到这个时候再继续运行以下指令。 

tar -jxf ../mpfr-3.1.0.tar.bz2
mv -v mpfr-3.1.0 mpfr
tar -Jxf ../gmp-5.0.4.tar.xz
mv -v gmp-5.0.4 gmp
tar -zxf ../mpc-0.9.tar.gz
mv -v mpc-0.9 mpc

GCC 文档推荐在源代码目录外创建目录编制 GCC： 

mkdir -v ../gcc-build
cd ../gcc-build

准备编译 GCC： 

../gcc-4.6.3/configure         \
    --target=$LFS_TGT          \
    --prefix=/tools            \
    --disable-nls              \
    --disable-shared           \
    --disable-multilib         \
    --disable-decimal-float    \
    --disable-threads          \
    --disable-libmudflap       \
    --disable-libssp           \
    --disable-libgomp          \
    --disable-libquadmath      \
    --enable-languages=c       \
    --without-ppl              \
    --without-cloog            \
    --with-mpfr-include=$(pwd)/../gcc-4.6.3/mpfr/src \
    --with-mpfr-lib=$(pwd)/mpfr/src/.libs

配置选项的含义：

--disable-shared

此开关强制 GCC 静态链接内部库以防止可能出现的有关宿主系统的问题。

--disable-decimal-float, --disable-threads, --disable-libmudflap, --disable-libssp, --disable-libgomp, --disable-libquadmath --disable-target-libiberty --disable-target-zlib

这些开关禁用十进制浮点扩展、线程、libmudflap、libssp 和 libgomp、libquadmath、libiberty 和 zlib 的支持。这些特性在编制交叉编译器时会编译失败，而且对于交叉编译临时的 libc 没有必要。

--disable-multilib

在 x86_64 上，LFS 还不支持多重库配置。此开关对 x86 是无害的。

--enable-languages=c

此选项确保只编制 C 编译器。这是目前需要的唯一的语言。

--without-ppl, --without-cloog

这些开关防止 GCC 依据宿主系统上可能出现但在 chroot 环境中不可用的 PPL 和 CLooG 库编制。

运行以下命令编译 GCC： 

make

编译现在完成了。通常我们现在要运行测试套件，但现阶段测试套件框架 (Tcl、Expect 和 DejaGNU) 还没有到位。在这里运行测试意义不大，因为这第一遍编译的程序很快会被第二遍的替换掉。

安装程序包： 

make install

使用 --disable-shared 意味着 libgcc_eh.a 文件不会创建和安装。Glibc 程序包因为在自己的编制系统中使用了 -lgcc_eh 而依赖这个库。此项依赖关系可以通过创建一个到 libgcc.a 的符号链接来满足，因为这个文件最后会包含通常包含在 libgcc_eh.a 中的对象。 

ln -vs libgcc.a `$LFS_TGT-gcc -print-libgcc-file-name | \
    sed 's/libgcc/&_eh/'`

有关此程序包的细节位于第 6.17.2 节 “GCC 的内容”。
