5.4. Binutils-2.22 - 第一遍

Binutils 程序包包含一个链接器、一个汇编器和其他用于处理对象文件的工具。

预计编制时间: 1 SBU
需求磁盘空间: 350 MB

5.4.1. 安装交叉 Binutils

注意

回去重新读一下上一节中的注记。读懂标记为重要的注记可以减少以后遇到的麻烦。

Binutils 作为第一个编译的程序包很重要，因为 GCC 和 Glibc 都会对汇编器和链接器进行特性测试来决定要启用禁用他们自己的哪些特性。

Binutils 文档推荐在源代码目录外创建目录编制 Binutils： 

mkdir -v ../binutils-build
cd ../binutils-build

注意

为了使本书余下部分列出的 SBU 值有意义，测量这个程序包从配置开始到安装结束的编制时间。要完成这一任务，像这样用 time 命令把这三个命令打包：time { ./configure ... && make&& make install; }。

注意

第五章的 SBU 值和需求磁盘空间不包含测试套件数据。

现在准备编译 Binutils： 

../binutils-2.22/configure --target=$LFS_TGT \
                           --prefix=/tools   \
                           --disable-nls     \
                           --disable-werror

配置选项的含义：

--target=$LFS_TGT

因为 LFS_TGT 变量中的计算机描述与 config.guess 脚本返回的值略有区别，此开关会告诉 configure 脚本调节 Binutil 的编制系统编制一个交叉链接器。

--prefix=/tools

此项告诉配置脚本准备安装 Binutils 程序于 /tools 目录中。

--disable-nls

此项禁用国际化，因为国际化在临时工具中不需要。

--disable-werror

此项防止编制由于宿主的编译器的警告而停止。

继续编译程序包： 

make

编译现在完成了。通常我们现在要运行测试套件，但现阶段测试套件框架 (Tcl、Expect 和 DejaGNU) 还没有到位。在这里运行测试意义不大，因为这第一遍编译的程序很快会被第二遍的替换掉。

如果在 x86_64 上编制，创建一个符号链接以确保工具链可靠： 

case $(uname -m) in
  x86_64) mkdir -v /tools/lib && ln -sv lib /tools/lib64 ;;
esac

安装程序包： 

make install

有关此程序包的细节位于第 6.13.2 节 “Binutils 的内容”。
