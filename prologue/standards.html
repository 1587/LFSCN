iv. LFS 与标准

LFS 的结构应尽可能符合 Linux 标准。基本标准有：

  • POSIX.1-2008

  • 文件系统层级标准 (FHS)

  • Linux 基础标准 (LSB) 核心要求 4.0

    LSB 有五个分立的标准：核心、C++、桌面、解释型语言和打印。除通用要求之外，还有一些架构特有的要求。LFS 试图符合前一节中提及的架构的相关要求。

    注意

    很多人不认可 LSB 的要求。定义它的主要目的是为了确保专有软件可以在兼容的系统环境下正确安装和运行。由于 LFS 是基于源代码的，用户对程序包有完全的控制，很多人选择不安装某些由 LSB 指定的程序包。

创建一个完整的 LFS 系统并通过 LSB 认证可以做到，但仍需很多 LFS 以外的程序包。多数附加程序包在 BLFS 中有安装引导。

LSB 需求且 LFS 支持的程序包

LSB 核心：

Bash、Binutils、Coreutils、Diffutils、File、Findutils、Gawk、Grep, Gzip, M4, Man-DB, Ncurses, Procps、Psmisc、Sed、Shadow、Tar、Util-linux、Zlib

LSB C++：

Gcc

LSB 桌面：

无

LSB 解释型语言：

Perl

LSB 打印：

无

LSB 多媒体：

无

LSB 需求且 BLFS 支持的程序包

LSB 核心：

Bc、Cpio、Ed、Fcrontab、Initd-tools、PAM、Sendmail (或 Postfix 或 Exim)

LSB C++：

无

LSB 桌面：

ATK、Cairo、Desktop-file-utils、Freetype、Fontconfig、Glib2、GTK+2、Icon-naming-utils、Libjpeg、Libpng、Libxml2、MesaLib、Pango、Qt3、Qt4、Xorg

LSB 解释型语言：

Python

LSB 打印：

CUPS

LSB 多媒体：

Alsa 库、NSPR、NSS、OpenSSL、Java

LSB 需求但 LFS 和 BLFS 都不支持的程序包

LSB 核心：

At、Batch、Lsb_release

LSB C++：

无

LSB 桌面：

无

LSB 解释型语言：

无

LSB 打印：

无

LSB 多媒体：

Xdg-utils
