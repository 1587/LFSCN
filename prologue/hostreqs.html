vii. 宿主系统需求

你的宿主系统需要安装不低于列出版本的如下软件。对多数的现代 Linux 发行版来说这应该不成问题。要注意很多发行版将软件头文件装入分立的程序包，通常以 “<程序包名称>-devel” 或 “<程序包名称>-dev” 的形式存在。如果你的发行版提供，请你安装。

比列出软件包更早的版本可能工作，但未被测试过。

  • Bash-3.2 (/bin/sh 应为到 bash 的符号或硬链接)

  • Binutils-2.17 (高于 2.22 的版本因为未经测试而不推荐)

  • Bison-2.3 (/usr/bin/yacc 应为到 bison 的链接或运行 bison 的脚本)

  • Bzip2-1.0.4

  • Coreutils-6.9

  • Diffutils-2.8.1

  • Findutils-4.2.31

  • Gawk-3.1.5 (/usr/bin/awk 应为到 gawk 的链接)

  • GCC-4.1.2 (高于 4.7.2 的版本因未经测试而不推荐)

  • Glibc-2.5.1 (高于 2.16.0 的版本因未经测试而不推荐)

  • Grep-2.5.1a

  • Gzip-1.3.12

  • Linux-2.6.25 (以 GCC-4.1.2 或更高版本编译)

    有内核版本需求的原因是根据开发人员推荐，在 第六章 编译 Glibc 时指定此版本。Udev 也要求此版本。

    如果宿主内核早于 2.6.25 或并非以 GCC-4.1.2 (或更高版本) 编译，你就需要用一个符合要求的内核来替换。有两种方法。其一，查看你的发行版是否提供 2.6.25 或更高版本的内核程序包。如果有，直接安装即可。如果没有，或者你不想安装，也可以自己编译一个。编译内核并配置引导加载器的方法 (请确保宿主系统使用 Grub) 位于 第八章。

  • M4-1.4.10

  • Make-3.81

  • Patch-2.5.4

  • Perl-5.8.8

  • Sed-4.1.5

  • Tar-1.18

  • Texinfo-4.9

  • Xz-5.0.0

注意上文提到的符号链接是按本书指令编制 LFS 必需的。指向其他软件 (比如 dash、mawk 等) 的符号链接可能工作，但未被 LFS 开发团队测试和支持，并可能与指令有偏差或需要某些程序包的额外补丁。

要查看你的宿主系统是否有所有正确版本并有能力编译程序，运行以下命令： 

cat > version-check.sh << "EOF"
#!/bin/bash
# Simple script to list version numbers of critical development tools

export LC_ALL=C
bash --version | head -n1 | cut -d" " -f2-4
echo "/bin/sh -> `readlink -f /bin/sh`"
echo -n "Binutils: "; ld --version | head -n1 | cut -d" " -f3-
bison --version | head -n1
if [ -e /usr/bin/yacc ];
  then echo "/usr/bin/yacc -> `readlink -f /usr/bin/yacc`"; 
  else echo "yacc not found"; fi

bzip2 --version 2>&1 < /dev/null | head -n1 | cut -d" " -f1,6-
echo -n "Coreutils: "; chown --version | head -n1 | cut -d")" -f2
diff --version | head -n1
find --version | head -n1
gawk --version | head -n1
if [ -e /usr/bin/awk ];
  then echo "/usr/bin/awk -> `readlink -f /usr/bin/awk`"; 
  else echo "awk not found"; fi

gcc --version | head -n1
ldd --version | head -n1 | cut -d" " -f2-  # glibc version
grep --version | head -n1
gzip --version | head -n1
cat /proc/version
m4 --version | head -n1
make --version | head -n1
patch --version | head -n1
echo Perl `perl -V:version`
sed --version | head -n1
tar --version | head -n1
echo "Texinfo: `makeinfo --version | head -n1`"
xz --version | head -n1

echo 'main(){}' > dummy.c && gcc -o dummy dummy.c
if [ -x dummy ] 
  then echo "gcc compilation OK";
  else echo "gcc compilation failed"; fi
rm -f dummy.c dummy
EOF

bash version-check.sh
