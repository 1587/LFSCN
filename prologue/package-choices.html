v. 使用本书中软件包的理由

就像前面所说，LFS的目的是构建一个完整可用的基础系统。这包括一些用于提供相对最小的工作环境的软件包，用户在此之上定制自己的系统。这些工具经常会用于替换他们本身。这并不意味着LFS是最小的操作系统。几个最为重要的软件包在其他系统上并不是必需的。下面列出了使用这些软件的理由。

  • Autoconf

    这个软件包包含可以用来根据模板生成 configure 脚本的程序。当源码改动的时候往往需要重新生成相应的 configure 脚本。

  • Automake

    这个软件包包含根据模板生成 Makefile 的程序。重新编制包的时候需要此程序。

  • Bash

    Bourne Shell。提供系统与用户的交互界面。可替换成其他 shell。因为其通用性和良好的可扩展性被选用。

  • Binutils

    包含汇编器、链接器等其他工具，用于处理对象文件。这个软件包用于编译 LFS 系统中的大部分软件。

  • Bison

    GNU 版 yacc，一个语法分析器生成器，用于构建其他几个 LFS 系统中的程序。

  • Bzip2

    用于压缩和解压 .bz2 文件。这个软件被用于解压大部分用于构建 LFS 系统的软件包。

  • Coreutils

    这个软件包包含一些用于操作和查看文件和文件夹的工具。被用于命令行文件管理。对 LFS 系统安装过程至关重要。

  • DejaGNU

    这个软件包包含一个用于测试其他程序的框架。只被安装在临时工具链中。

  • Diffutils

    提供用于比较文件和文件夹不同之处。可以用来生成 patch，并在许多软件的安装流程中被使用。

  • Expect

    用于测试其他程序。只被安装在临时工具链中。

  • E2fsprogs

    包含用于处理 ext2、ext3、ext4 文件系统的工具。这些文件系统在 linux 系统中被广泛使用。

  • File

    用于检测文件类型。几个软件包的编译需要他。

  • Findutils

    提供用于在文件系统中查找文件的工具。被许多安装脚本调用。

  • Flex

    词法分析器生成器。GNU 版 lex。被用于一些软件包。

  • Gawk

    awk 脚本解释器。被安装脚本调用

  • Gcc

    GNU 编译器集，包含用于编制 LFS 系统的 C 与 C++ 编译器。

  • GDBM

    GNU 数据管理库。被Man-DB需要。

  • Gettext

    包含国际化和本地化的库和工具。

  • Glibc

    标准 C 库，缺了他其他程序基本无法工作。

  • GMP

    包含数学库提供任意精度计算。用于 Gcc 的构建。

  • Grep

    用于在文件中查找。大多数安装脚本使用了 grep。

  • Groff

    包含用于处理和格式化文本的工具。一个重要的功能是格式化 man 手册页。

  • GRUB

    用于引导系统。

  • Gzip

    压缩和解压缩 .gz 文件。用于解压缩其他软件包。

  • Iana-etc

    提供一些网络服务和协议的数据。用于提供网络功能。

  • Inetutils

    提供基础的网络管理程序。

  • IProute2

    包含管理基本的和高级的 IPv4 和 IPv6 网络。因为 IPv6 功能被选用

  • Kbd

    提供键盘映射表，键盘相关的工具用于非标准 US 键盘。还有一些控制台用到的字体。

  • Kmod

    包含用于管理 Linux 内核模块的程序。

  • Less

    用于浏览文件，被 Man-DB 使用。

  • Libtool

    用于构建库

  • Linux Kernel

    操作系统内核。GNU/Linux 环境的 Linux 。

  • M4

    通用文本宏处理器，用作其他程序包的编制工具。

  • Make

    指导程序包编制。被 LFS 中几乎每个程序包需求。

  • Man-DB

    包含用于查找和查看 man 手册页的工具。使用该软件而不是 man 是由于他更好的国际化支持。

  • Man-pages

    提供 Linux 系统 man 手册页的内容。

  • MPC

    提供复杂数学计算支持。Gcc 安装时需要。

  • MPFR

    提供多精度数学支持。Gcc 安装时需要。

  • Ncurses

    提供用于控制字符界面的函数库。经常用于字符菜单。被 LFS 系统中的一些软件使用。

  • Patch

    根据 patch 修正或创建文件。patch一般由 diff 程序创建。在 LFS 系统一些软件的安装流程中用到。

  • Perl

    包含 Perl 语言的解释器。被一些 LFS 系统中的软件的安装或测试流程用到。

  • Procps

    提供用于监视系统进程的工具。这些工具对系统管理非常有用，并且被 LFS 系统启动脚本使用。

  • Psmisc

    提供用于显示系统中正在运行的进程的信息的工具。对系统管理非常有用。

  • Readline

    提供一系列函数库，用于提供命令行编辑和历史查询功能。被 Bash 使用。

  • Sed

    流文本编辑器。被 LFS 系统大部分软件包的 configure 脚本使用。

  • Shadow

    提供更安全的系统密码存储。

  • Sysklogd

    提供用于记录系统日志的程序。

  • Sysvinit

    提供 init 程序。该进程是 Linux 系统中所有进程的父进程。

  • Tar

    压缩包管理。

  • Tcl

    提供 Tcl 语言 (Tool Command Language) 支持。在一些测试中用到。只被安装在临时工具链。

  • Texinfo

    提供 info 页读取、写入、转换的相关程序。在许多 LFS 系统软件包的安装流程中用到。

  • Udev

    提供一个在用户空间的守护进程，用于动态创建设备节点。

  • Util-linux

    提供一些实用工具用于系统，硬盘分区，收集信息。

  • Vim

    Vim 编辑器。可根据个人喜好选择其他编辑器。

  • XZ Utils

    提供 XZ 或 LZMA 压缩包格式的支持。

  • Zlib

    为一些程序提供压缩解压支持。
