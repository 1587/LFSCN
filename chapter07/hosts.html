7.3. 自定义 /etc/hosts 文件

如果要配置一个网卡，选定 IP 地址、FQDN 和用于 /etc/hosts 文件的可用对应关系。语法为： 

IP_address myhost.example.org aliases

除非计算机对互联网可见 (例如有注册的域和可用的 IP 地址块——多数用户没有)，确保 IP 地址在私有 IP 地址区间中。可用的区间包括： 

Address Range                   Normal Prefix
10.0.0.1 - 10.255.255.254       8
172.x.0.1 - 172.x.255.254       16
192.168.y.1 - 192.168.y.254     24

x 可以是 16-31 的数值，y 可以是 0-255 的数值。

一个可用的私有 IP 地址可以是 192.168.1.1。对于此 IP 地址可用的 FQDN 可以是 lfs.example.org。

即使没有使用网卡，有些程序也需要一个可用的 FQDN 才能正确工作。

运行以下命令创建 /etc/hosts 文件： 

cat > /etc/hosts << "EOF"
# Begin /etc/hosts (network card version)

127.0.0.1 localhost
<192.168.1.1> <HOSTNAME.example.org> [alias1] [alias2 ...]

# End /etc/hosts (network card version)
EOF

<192.168.1.1> 和 <HOSTNAME.example.org> 的值应该按照指定的用途或者需求进行修改 (如果由网络/系统管理员分配了一个 IP 地址并且计算机要链接到已经存在的网络)。可选的对应名称可以忽略。

如果不配置网卡，运行以下命令创建 /etc/hosts 文件： 

cat > /etc/hosts << "EOF"
# Begin /etc/hosts (no network card version)

127.0.0.1 <HOSTNAME.example.org> <HOSTNAME> localhost

# End /etc/hosts (no network card version)
EOF
