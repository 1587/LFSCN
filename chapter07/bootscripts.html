7.6. LFS-Bootscripts-20130123

LFS-Bootscripts 程序包包含一组脚本用于在引导/关机时启动/停止 LFS 系统。

预计编制时间：少于 0.1 SBU
需求磁盘空间：260 KB

7.6.1. 安装 LFS-Bootscripts

安装程序包： 

make install

7.6.2. LFS-Bootscripts 的内容

安装的脚本：checkfs、cleanfs、console、functions、halt、ifdown、ifup、localnet、modules、mountfs、mountkernfs、network、rc、reboot、sendsignals、setclock、static、swap、sysctl、sysklogd、template、udev 和 udev_retry
安装的目录：/etc/rc.d、/etc/init.d (符号链接)、/etc/sysconfig、/lib/services、/lib/lsb (符号链接)

简介

checkfs

在挂载文件系统之前检查其完整性 (但日志和基于网络的文件系统除外)

cleanfs

移除重新引导时不应保留的文件，例如 /var/run/ 和 /var/lock/ 中的文件；它重新创建 /var/run/utmp 并移除可能存在的 /etc/nologin、/fastboot 和 /forcefsck 文件

console

为选中的键盘布局加载正确的的键盘映射表并设置屏幕字体

functions

包含多个引导脚本的共用功能，例如错误和状态检查

halt

终止系统运行

ifdown

停止一个网络设备

ifup

初始化一个网络设备

localnet

设置系统的主机名和本地回环设备

modules

加载在 /etc/sysconfig/modules 中列出的内核模块，使用其中给定的参数

mountfs

挂载所有的文件系统，但标记为 noauto 或基于网络的除外

mountkernfs

挂载虚拟内核文件系统，例如 proc

network

设置网络接口，例如网卡，并且设置默认网关 (如果可用)

rc

主要的运行级别控制脚本；负责逐一运行所有其他的引导脚本，序列由处理的符号链接名称产生

reboot

重新引导系统

sendsignals

确保每个进程在系统终止或重新引导之前都结束

setclock

如果硬件时钟没有设为 UTC 时间则重设内核时钟为本地时间

static

提供给网络接口分配静态 IP 地址的功能

swap

启用和禁用交换文件和分区

sysctl

如果存在，从 /etc/sysctl.conf 中加载系统配置的值到内核中

sysklogd

启动和停止系统和内核日志 daemon

template

一个模板，用于给其他 daemon 创建自定义引导脚本

udev

准备 /dev 目录并启动 Udev

udev_retry

重试失败的 uevent，如果需要则将产生的规则文件复制到 /etc/udev/rules.d
