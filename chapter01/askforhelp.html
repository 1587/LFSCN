1.5. 请求帮助

如果根据本书的示例进行安装时发生了任何错误，请检查常见问题列表 http://www.linuxfromscratch.org/faq/#generalfaq。大多数问题会在这里解决。如果你的问题在这里没有解决，试着自己找到错误信息。下面的提示会在纠错中提供帮助：http://www.linuxfromscratch.org/hints/downloads/files/errors.txt。

如果你的问题不能在常见问题列表中找到答案，请查找邮件列表 http://www.linuxfromscratch.org/search.html。

我们还有强大的 LFS 社区，愿意通过邮件列表和 IRC 提供帮助 (参考本书中 第 1.4 节 “相关资源”)。然而，我们每天都会收到几个支持问题，其中有很多都只需查看 FAQ 或搜索邮件列表即可回答。所以，为了我们能提供最大的帮助，你要自己先做一点调查。这样我们就能专注于更不寻常的支持需求。如果你的搜索没有产生解决方案，请在帮助请求中包含以下所有信息。

1.5.1. 需要提及

简要描述遇到的问题，并提供以下内容：

  • 使用的本书版本 (此处为 SVN-20120827)

  • 用于创建 LFS 的宿主发行版和版本

  • 第 vii 节 “宿主系统需求” 的输出

  • 出现问题的程序包或章节

  • 错误消息或症状

  • 注明是否从本书偏离过。

注意

偏离本书不代表我们不会帮你。无论如何，LFS 都有关个人喜好。走在改动的第一线会帮助我们估计你的问题的可能原因。

1.5.2. 配置脚本问题

如果在运行 configure 脚本时出现问题，检查 config.log 文件。此文件可能包含在运行 configure 期间出现的未印出至屏幕的问题。请求帮助时包含相关行的内容。

1.5.3. 编译问题

寻求编译问题的原因时屏幕输出和几个文件的内容会很有用。configure 脚本和 make 运行过程中的屏幕输出很有帮助。不必包含整个输出，但是一定要包含足够的相关信息。以下是 make 的屏幕输出要包含的信息类型： 

gcc -DALIASPATH=\"/mnt/lfs/usr/share/locale:.\"
-DLOCALEDIR=\"/mnt/lfs/usr/share/locale\"
-DLIBDIR=\"/mnt/lfs/usr/lib\"
-DINCLUDEDIR=\"/mnt/lfs/usr/include\" -DHAVE_CONFIG_H -I. -I.
-g -O2 -c getopt1.c
gcc -g -O2 -static -o make ar.o arscan.o commands.o dir.o
expand.o file.o function.o getopt.o implicit.o job.o main.o
misc.o read.o remake.o rule.o signame.o variable.o vpath.o
default.o remote-stub.o version.o opt1.o
-lutil job.o: In function `load_too_high':
/lfs/tmp/make-3.79.1/job.c:1565: undefined reference
to `getloadavg'
collect2: ld returned 1 exit status
make[2]: *** [make] Error 1
make[2]: Leaving directory `/lfs/tmp/make-3.79.1'
make[1]: *** [all-recursive] Error 1
make[1]: Leaving directory `/lfs/tmp/make-3.79.1'
make: *** [all-recursive-am] Error 2

这一例中很多人只会包含最下面一部分。 

make [2]: *** [make] Error 1

这些信息不足以诊断问题，它只是说有东西出错了，而不是什么出错了。上面例子中的整段输出都要保留下来，它包含了运行的指令和相关的错误消息。

关于如何在网络上提问的一篇好文章位于 http://catb.org/~esr/faqs/smart-questions.html 。阅读并遵照其中提示会提升获得帮助的可能性。
