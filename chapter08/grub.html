8.4. 使用 GRUB 来设置启动流程

8.4.1. 介绍

警告

错误配置 GRUB 会造成你的系统无法操作（启动），除了使用其他的可启动媒介，例如光盘。这一章节不要求你启动你的LFS系统，你只需要修正系统当前的 Boot Loader，像 Grub-Legacy, GRUB2, 或者 LILO。

确保你有一个应急的可引导磁盘（译者注，光盘Linux也行），这使我们可以在电脑不能启动时进行“救援”。如果你没有，可以创建一个。为此，你需要跳到BLFS中，安装 xorriso。 

cd /tmp &&
grub-mkrescue --output=grub-img.iso &&
xorriso -as cdrecord -v dev=/dev/cdrw blank=as_needed grub-img.iso

8.4.2. GRUB 的名称转换

GRUB 使用自己的命名方式来标识硬盘和分区， 比如 (hdn,m)，n是硬盘的编号，m 是分区的编号。硬盘的编号从零开始，分区的编号是从1开始（扩展分区的编号从5开始）。注意这与早期的使用0开始的软件版本不同。例如，分区sda1 对于GRUB是 (hd0,1) GRUB，sdb3是 (hd1,3)。与Linux不同的是，GRUB 并不将光驱当作硬盘。例如，如果使用一个光驱作为hdb， 另一个硬盘作为 hdc，那么这个硬盘将是(hd1)。

你可以用下面的方法查看GRUB是如何看待你的硬件的： 

grub-mkdevicemap --device-map=device.map
cat device.map

8.4.3. 创建配置

GRUB 的工作方式是将信息写入硬盘的开头区域。这个区域不属于任何一个文件系统。这个区域储存的程序会在启动时读取启动分区的GRUB模块。默认位置为 /boot/grub/。

启动分区的位置是用户的可选配置之一。一个建议是使用一个单独的小分区（建议100M）来作为启动分区。如果这样，你的机子上所有的系统（LFS 系统和一些发行版）都可以读取这个分区上相同的配置文件。如果你打算这么做，你需要挂载这个分区，然后将 /boot 文件夹 （包括你在前面章节编译的的内核）中的东西移动到这个被挂载了的分区。然后卸载这个分区，再重新将其挂载在 /boot，并确认你更新了 /etc/fstab。

使用当前的LFS分区一样没问题， 只是当你配置多系统启动的时候会有些麻烦。

根据上面的信息，决定你的根分区（或者启动分区，如果你选择使用一个单独的启动分区）的设计。下面的例子，假设根分区（或者单独的启动分区）是 sda2。

安装 GRUB 到/boot/grub 并设置引导：

警告

下面的命令会覆盖原先的boot loader。如果你不想如此， 不要这么做。例如，如果你使用一个第三方的启动管理器来管理MBR。 

grub-install /dev/sda

注意

grub-install 是一个脚本，它会调用 grub-probe。这个程序可能会失败，并提示 "cannot stat `/dev/root'"。如果这样，创建一个从你的根分区到 /dev/root 的临时符号链接： 

ln -sv /dev/sda2 /dev/root

这个符号链接仅仅会存在于下次重启。它制作用于安装流程。

8.4.4. 创建配置文件

创建 /boot/grub/grub.cfg： 

cat > /boot/grub/grub.cfg << "EOF"
# Begin /boot/grub/grub.cfg
set default=0
set timeout=5

insmod ext2
set root=(hd0,2)

menuentry "GNU/Linux, Linux 3.2.11-lfs-SVN-20120314" {
        linux   /boot/vmlinuz-3.2.11-lfs-SVN-20120314 root=/dev/sda2 ro
}
EOF

注意

从 GRUB 的角度来看内核文件仅与其所在的分区相关。如果你使用一个单独的启动分区，从 linux 那行删掉/boot。你同时也需要将 set root 改成你的根分区。

GRUB是一个很棒的软件，它提供了大量的选项用于不同的设备，操作系统，分区。它也有一些个性化的设置如图形启动界面，启动声音，鼠标输入等等。这些话题超出了这篇介绍的范围。

小心

grub-mkconfig这个命令可以自动生成一个配置文件。他会调用一系列 /etc/grub.d/ 中的脚本，并覆盖你原先的配置文件。这些脚本起初是为“非源码”发行版准备的，并不建议在LFS系统中使用它们。如果你使用一个商业Linux发行版，那么你有很大机会会用到他们。确保你备份了 grub.cfg 文件。
