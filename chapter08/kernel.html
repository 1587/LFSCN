8.3. Linux-3.6.5

Linux 程序包包含 linux 内核。

预计编制时间：1.0 - 5.0 SBU
需求磁盘空间：540 - 800 MB

8.3.1. 安装内核

编制内核包含一些步骤——配置，编译和安装。阅读内核源代码树中的 README 文件来获知不同于本书所述的配置内核方法。

运行以下命令来准备编译： 

make mrproper

这确保内核树完全干净。内核开发组建议先于每个内核编译运行此命令。不要期望内核树在 un-tarring 后保持干净。

经由一个目录驱动的接口配置内核。查阅 http://www.linuxfromscratch.org/hints/downloads/files/kernel-configuration.txt 来获得内核配置的常规信息。 BLFS 中有关某些需求 LFS 不包含的程序包的内核配置的一些信息可在 http://www.linuxfromscratch.org/blfs/view/svn/longindex.html#kernel-config-index 找到。

注意

由于近期在 udev 中的改动，确保选择了： 

Device Drivers  --->
  Generic Driver Options  --->
     Maintain a devtmpfs filesystem to mount at /dev

make LANG=<host_LANG_value> LC_ALL= menuconfig

make 参数的含义：

LANG=<host_LANG_value> LC_ALL=

这建立区域设置为宿主所使用的。这对一个正确的 menuconfig ncurses 界面线在 UTF-8 linux 文本控制台上绘制是必须的。

确保替换 <host_LANG_value> 为来自你的 host 的 $LANG 变量值。如果没有设置，你可以使用 host 的 $LC_ALL 或 $LC_CTYPE 的值。

可以替代地， make oldconfig 在一些场合可能更加合适。查阅 README 文件来获得更多信息。

如果需要，通过拷贝宿主系统的内核配置文件 .config (这里默认它可用) 到未打包的 linux-3.6.5 目录，来跳过内核配置。但是我们不推荐如此做。探索整个配置菜单并白手起家创建一个内核配置通常更好。

编译内核镜像和模块： 

make

如果使用内核模块，/etc/modprobe.d 中的模块配置可能被需要。关于模块和内核配置的信息位于 第 7.4 节 “LFS 系统上的设备和模块管理” 和 linux-3.6.5/Documentation 目录的内核文档中。同时， modprobe.conf(5) 可能包含需要的信息。

如果内核配置需要，安装这些模块： 

make modules_install

在内核编译完成后，需要额外的步骤来完成安装。一些文件需要被拷贝到 /boot 目录。

到内核镜像的路径可能根据所使用的平台而改变。下面的文件名可根据你的喜好改变，但文件名的主干必须是 vmlinuz 以和下个部分中的提及的引导进程自动安装相兼容。以下的命令默认在 x86 架构上运行： 

cp -v arch/x86/boot/bzImage /boot/vmlinuz-3.6.5-lfs-SVN-20121102

System.map 是内核的一个符号文件。它映射内核 API 中的每个函数的函数入口点以及正在运行的内核的内核数据结构地址。它被用作调查内核问题时的一个资源。运行以下命令来安装映射文件： 

cp -v System.map /boot/System.map-3.6.5

由上面的 make menuconfig 步骤生成的内核配置文件 .config 包含所有刚刚被编译的内核配置选项。保存此文件以备日后查阅是一个好主意： 

cp -v .config /boot/config-3.6.5

安装 Linux 内核文档： 

install -d /usr/share/doc/linux-3.6.5
cp -r Documentation/* /usr/share/doc/linux-3.6.5

注意内核源目录中的文件不属于 root 是很重要的。每当一个程序包被以用户 root 解压 (就像我们在 chroot 中做的那样) ，文件得到打包者的电脑上的用户和组 ID 。这对其他任何要安装的程序包通常不成问题，鉴于源代码树在安装后移除。然而 Linux 源代码树通常会保存较长的时间。因此，打包者使用的 ID 有机会被赋值到机器上的某处。那个人将会得到修改内核源代码的权限。

如果内核源代码树要被保存，在 linux-3.6.5 目录运行 chown -R 0:0 来确保所有文件被用户 root 所有。

警告

一些内核文档建议创建一个从 /usr/src/linux 指向内核源代码目录的符号链接。这是针对 2.6 之前版本内核的并且 一定不要 在一个 LFS 系统上创建，因为它可能在你在基本 LFS 系统完成后试图编制程序包时造成问题。

警告

系统的 include 目录下的头文件应该 始终 和 Glibc 一起编译，也就是说，来自这个 Linux 系统内核中 tarball 的纯净头文件。这样一来，它们应当 永不 被原始的内核头文件或任何其他内核纯净头文件替换。

8.3.2. 配置 Linux 模块加载序列

/etc/modprobe.d/usb.conf 需要被创建。这样一来，如果 USB 驱动 (ehci_hcd， ohci_hcd 和 uhci_hcd) 已被编制为模块，他们会按正确的序列被加载；ehci_hcd 需要先于 ohci_hcd 和 uhci_hcd 被加载以避免引导时输出警告。

运行以下命令来创建一个新 /etc/modprobe.d/usb.conf 文件： 

install -v -m755 -d /etc/modprobe.d
cat > /etc/modprobe.d/usb.conf << "EOF"
# Begin /etc/modprobe.d/usb.conf

install ohci_hcd /sbin/modprobe ehci_hcd ; /sbin/modprobe -i ohci_hcd ; true
install uhci_hcd /sbin/modprobe ehci_hcd ; /sbin/modprobe -i uhci_hcd ; true

# End /etc/modprobe.d/usb.conf
EOF

8.3.3. Linux 的内容

安装的文件：config-3.6.5、vmlinux-3.6.5-lfs-SVN-20121102-3.6.5 和 System.map-3.6.5
安装的目录：/lib/modules、/usr/share/doc/linux-3.6.5

简介

config-3.6.5

包含所有内核配置选项

vmlinux-3.6.5-lfs-SVN-20121102

Linux 系统的引擎。当开启电脑时，内核是运行系统中首先被加载的部分。它检测并初始化电脑硬件的所有组件，然后让这些组件以一个文件树的形式对软件可用并转换一个单个的 CPU 为可以看上去是同时地运行大量程序的多线程机

System.map-3.6.5

一个地址和符号的列表；它映射内核中所有函数和数据结构的入口点和地址
