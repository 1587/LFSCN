2.3. 在新分区上创建文件系统

现在创建好了空白分区，就可以创建文件系统了。Linux 中最常用的是 ext2 文件系统，但有了新的大容量硬盘，日志文件系统越来越受欢迎。ext3 文件系统是 ext2 的一种广泛应用的增强，加入了日志兼容性并且与 E2fsprogs 工具兼容。我们将创建 ext3 文件系统。创建其他文件系统的指令位于 此处。

要在 LFS 分区上创建 ext3 文件系统，运行以下命令： 

mke2fs -jv /dev/<xxx>

将 <xxx> 替换为 LFS 分区名 (前面的例子中是hda5)。

注意

有些发行版在他们的文件系统创建工具 (E2fsprogs) 中使用了自定义特性，这可能导致在第九章引导到新的 LFS 系统时遇到麻烦，因为 LFS 安装的 E2fsprogs 不支持这些特性；你会收到类似 “unsupported filesystem features, upgrade your e2fsprogs” 的错误消息。要检查你的宿主系统是否使用了自定义增强，运行以下命令： 

debugfs -R feature /dev/<xxx>

如果输出包含除 has_journal、ext_attr、resize_inode、dir_index、filetype、sparse_super、large_file 或 needs_recovery 之外的选项，你的宿主就可能有自定义增强。这种情况下，为了防止以后出问题，你应该编译一下 E2fsprogs 程序包并使用产生的二进制来重新创建 LFS 分区上的文件系统： 

cd /tmp
tar -xzvf /path/to/sources/e2fsprogs-1.42.5.tar.gz
cd e2fsprogs-1.42.5
mkdir -v build
cd build
../configure
make #not make install
./misc/mke2fs -jv /dev/<xxx>
cd /tmp
rm -rfv e2fsprogs-1.42.5

如果你要使用已存在的 swap 分区，就不必再创建一个。如果创建了新的 swap 分区，需要用以下命令初始化之： 

mkswap /dev/<yyy>

将 <yyy> 替换为 swap 分区名。
